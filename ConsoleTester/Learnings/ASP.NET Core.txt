[完了] LogLevel.None について書く

    通常、None == 0 とするが、LogLevel.None == 6 になっている。

    これは MS による enum のガイドラインに反するし、default で得られる (LogLevel) 0 が Trace なのは違和感がある。

    しかし、「○○以上をログ」というふるい分けなので、実装としては理由がある。「None 以上をログ」で6になると、Critical の5でも (5 >= 6) == false になって出力されない。

        Connection start, stop, and keep alive responses. HTTPS certificate information.
    Microsoft.AspNetCore.StaticFiles
        Files served.
    Microsoft.EntityFrameworkCore
        General Entity Framework Core diagnostics. Database activity and configuration, change detection, migrations.

                "Microsoft.Hosting.Lifetime": "Information"

        }

[完了] ウェブ系の開発を学ぶ playground を追加

    Tester というほど完成度が高くならない。英語圏では playground という表現が散見される。「最初はムチャクチャでっせ」のニュアンスで。

    --------------------------------------------------------------------------------

    最近の開発における一つの問題は、フレームワークをゴテゴテと盛るのが普通になっていて、一つ入れるごとに数十のファイルが流し込まれること。playground に10も20もフレームワークを入れれば、そのプロジェクトのファイル数はエラいことになる。

    たとえば、JavaScript 系のライブラリーについては、Angular が無事に死に、Vue.js と React はやはり強く、Svelte は、まだあまり知られていないが、Twitter で見る限り使った人の評価は高いようで、次のページの表にはないが、Nuxt.js というのもちょくちょく見る。

    React vs Angular vs Vue: Which Framework to Choose in 2022
    https://2muchcoffee.com/blog/react-vs-angular-vs-vue-which-framework-to-choose-in-2022/

    こういうのを playground に一通り入れ、「Vue.js でレンダリングしてみたら」「React なら」といったことをページの切り換えにより全て試せるようにするなら、これは Nekote の一部でなく、独立したソリューションであるべき。

    --------------------------------------------------------------------------------

    どうせやるなら、本番の開発においてコピペできるくらい、きちんと練り上げられたコードを集めたい。となると、それをそのまま Twitter で、画像やビデオを付けて紹介したい気持ちも生じてくる。そういうことをした方が、もっと詳しい人からダメ出しなどをもらえる時代。
