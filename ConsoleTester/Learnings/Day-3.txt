となると、メモやコメントの英語化を考えるわけだが、これは Google Translate や DeepL の API に頼ろうと思う
自分は、仕事も普段の生活も100％近く英語なので英語での読み書きにほとんど困らないが、
    コードのコメントを書くにおいて「ここに the は必要か」だとか、「これは複数形の方がいいのでないか」だとかの、
    ネイティブでも「口に出してみて口に判断してもらう」的なことを非ネイティブの自分が理屈で1秒も考えたくないので全てを日本語で書いている
行コメントも多いため句点を省く独特の書き方だが、文が複数行になるなら2行目以降にインデントを付けていて、句点を付けるのは自動化できる
ということは、段落コメントや行コメントをプログラムで検出し、それぞれを API で訳したものとセットでデータベースに入れることで、
    NekoteTranslated レポジトリーに、メモもコメントも英語に翻訳されたクローンを用意することは、許容範囲内のコストで可能だろう

実際には、検出されたメモやコメントを段落・行ごとに上に表示し、下に訳文を表示し、訳文が壊れていれば上を修正して再翻訳するようなワークフローになる
複数行のコメントには必ず // を使い、行中コメントのみに /* ... */ を使っているので、アプリで修正した原文を元のファイルに戻すことは100％に近い精度で可能
原文にネタを入れているなどで翻訳できないなら、訳文の方を自分で書けばよい
TextBox の内容が変更されれば自動的に Edited フラグが立ち、
    そのことは NekoteTranslated 側には出力されないが、アプリ内では区別されるように
コードは共通で、日本語版で施された更新は必ず英語版に反映される
他者によるコミットを認める考えは今のところなく、英語版の方が勝手に進むことはない
それで Twitter には英語版の差分の URL を貼れ、エンゲージメントに期待できる
コスト評価がまだだが、ワークフローとしては問題がなさそう
